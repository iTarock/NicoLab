# Security

## Baseline

NicoLab uses a practical baseline: a small set of rules, applied consistently.

## Main controls

| Control | Status | Notes |
| --- | --- | --- |
| MFA on critical accounts | To fill | GitHub, Cloudflare, NAS |
| Password manager | To fill | One secret, one service |
| Tested backups | To fill | Track last restore date |
| Public services reviewed | To fill | Explain why exposure is needed |
| Scheduled updates | To fill | Frequency and rollback |
| Minimum logs | To fill | Access log, error log, alerts |

## Golden rule

A service exposed without a reason, without logs and without backups is not "online": it is technical debt.
