# Contributing

Thanks for taking an interest in NicoLab. This repository is public documentation for a real home lab, so contributions should improve clarity without exposing sensitive operational details.

## Good Contributions

- Fixing unclear language.
- Improving diagrams, flow descriptions or runbooks.
- Suggesting safer defaults.
- Adding public-safe examples.
- Reporting broken links or rendering problems.

## Before Opening a Pull Request

1. Keep all documentation in English.
2. Do not include real secrets, credentials, tokens, private keys or recovery codes.
3. Avoid publishing exact internal addresses, firewall rules or hostnames.
4. Prefer placeholders such as `service.example.com`, `10.0.0.0/24` and `REDACTED`.
5. Keep changes focused and easy to review.

## Documentation Style

- Use short sections with clear headings.
- Use tables for inventories and controls.
- Use Mermaid for flows when possible.
- Write runbooks as action lists.
- Explain why a decision was made, not only what changed.
