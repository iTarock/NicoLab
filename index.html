Flow mapping

Understand the path first. Then decide what deserves exposure.

Each flow explains where traffic enters, which controls it crosses, which service it reaches, and what logs or backups are needed when something breaks.

Security baseline

A simple checklist, serious enough to prevent chaos.

✓

Asset inventory

Services, ports, domains, handled data, owner and operating status.

✓

Least privilege

Separate accounts, minimum access, MFA where possible and a password manager.

✓

Patch rhythm

A routine for updates, rollback and change notes.

✓

Backup proof

Having backups is not enough: restore testing needs to be tracked.

✓

Exposure review

Every public service needs a reason and protection in front of it.

✓

Incident notes

Short runbooks for what to do when a service does not respond.

Repository content

The documentation grows like the lab: small, useful and versioned.

The included Markdown files are the base for publishing decisions, procedures and mappings. Screenshots, Mermaid diagrams and changelogs can be added without changing the site structure.
