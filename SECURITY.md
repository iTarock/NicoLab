# Security Policy

NicoLab documents a real lab environment through public-safe examples. Security reports are welcome and appreciated.

## Reporting a Security Issue

Do not create a public GitHub issue for exposed secrets, private infrastructure details, authentication bypasses, accidental publication of sensitive data or vulnerabilities in a live service.

Use a private contact channel instead:

- Email: `support@itarock.ch`

## Public-Safe Disclosure

Reports should include what is affected, why it matters, how to reproduce the issue safely and suggested remediation if available.

## Scope

In scope: this repository, the static NicoLab site and documentation that may reveal sensitive operational details.

Out of scope: denial-of-service testing, social engineering, attacks against third-party providers and attempts to access private services without explicit permission.
